Инфраструктура развернута в "серой" (внутренней) сети с выходом в интернет через 
Floating IP (FIP).

Внешний доступ: Трафик поступает на публичный адрес 185.102.75.187, привязанный к балансировщику.

Внутренняя сеть: Все узлы (балансировщик и веб-серверы) общаются между собой в изолированном сегменте.

Источник конфигурации: Логика сетевого взаимодействия и базовой настройки ОС основана на документации PS Cloud: 
https://static-docs.archive.pscloud.io/openstack_centos7_gray_network_fip.txt

Структура репозитория
PS/
├── ansible/             # Автоматизация настроек
│   ├── roles/           # Роли (apache, haproxy)
│   ├── site.yml         # Главный плейбук
│   └── hosts.ini        # Инвентарь серверов
├── terraform/           # Описание инфраструктуры
│   └── main.tf          # Ресурсы облака
├── .gitignore           # Игнорируемые файлы (ключи, стейты)
└── README.md            # документация

Прежде чем запускать команды, нужно убедиться, что Terraform установлен. 
Инициализация  
Скачиваем необходимые драйверы (провайдеры) для работы с облаком PS Cloud:
Bash
terraform init

Планирование 
Проверяем, какие ресурсы будут созданы (серверы, сети, балансировщик):
Bash
terraform plan

Применение 
Создаем инфраструктуру в облаке:
Bash
terraform apply
(Введите yes для подтверждения)

Ansible 
 Настройка серверов
Когда серверы созданы и доступны по сети, переходим к их конфигурации. Убедитесь, что вы находитесь в корне проекта или папке ansible/.
Проверка доступности  
Убедитесь, что Ansible видит все узлы из вашего файла hosts.ini:
Bash
ansible all -m ping -i hosts.ini
Запуск настройки 
Запускаем основной сценарий (плейбук), который установит Apache, настроит HAProxy и применит политики безопасности:
Bash
ansible-playbook -i hosts.ini site.yml
