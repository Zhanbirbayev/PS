Infrastructure as Code: Web-Cluster with Load Balancer
Данный проект реализует автоматизированное развертывание отказоустойчивой инфраструктуры в облаке.

Сеть: Инфраструктура развернута в «серой» (внутренней) сети с выходом в интернет через Floating IP (FIP).

Внешний доступ: Трафик поступает на публичный адрес *********, привязанный к балансировщику HAProxy.

Внутренняя сеть: Все узлы (балансировщик и веб-серверы Apache) общаются между собой в изолированном сегменте для обеспечения безопасности.

Источник конфигурации: Логика сетевого взаимодействия основана на документации PS Cloud.
https://static-docs.archive.pscloud.io/openstack_centos7_gray_network_fip.txt

Перед началом убедитесь, что установлены переменные окружения для доступа к OpenStack (через GitHub Secrets или .env).

1. Инициализация
Скачивание необходимых провайдеров для работы с PS Cloud:
terraform init

3. Планирование
Проверка ресурсов, которые будут созданы (серверы, сети, правила доступа)
terraform plan

3. Применение
Создание ресурсов в облаке:
terraform apply 

Конфигурация серверов (Ansible)
После того как Terraform поднял серверы, Ansible берет на себя установку софта.

Безопасность и Секреты
Проект использует GitHub Actions Secrets для защиты конфиденциальных данных. В плейбуке используются следующие переменные:

s3_host: Адрес S3 хранилища (**********).

s3_username: Логин для доступа к MinIO.

s3_password: Пароль для скачивания ролей.

1. Проверка доступности
Проверяем SSH-соединение со всеми узлами:
ansible all -m ping -i hosts.ini

3. Запуск настройки (Fetch & Deploy)
Основной сценарий автоматически скачивает роли из S3 хранилища, распаковывает их и настраивает кластер:
ansible-playbook -i hosts.ini site.yml

Результат
Apache: 3 узла с персонализированными страницами Welcome to <hostname>.

HAProxy: Балансировка трафика
