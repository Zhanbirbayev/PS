Infrastructure as Code: Web-Cluster with Load Balancer
Данный проект реализует автоматизированное развертывание отказоустойчивой инфраструктуры в облаке PS Cloud.

🏗 Архитектура
Сеть: Инфраструктура развернута в «серой» (внутренней) сети с выходом в интернет через Floating IP (FIP).

Внешний доступ: Трафик поступает на публичный адрес 185.102.75.187, привязанный к балансировщику HAProxy.

Внутренняя сеть: Все узлы (балансировщик и веб-серверы Apache) общаются между собой в изолированном сегменте для обеспечения безопасности.

Источник конфигурации: Логика сетевого взаимодействия основана на документации PS Cloud.

📂 Структура репозитория
Bash

PS/
├── ansible/               # Автоматизация конфигурации ПО
│   ├── roles/             # Роли (apache, haproxy), подгружаемые из S3
│   ├── site.yml           # Главный плейбук (Workflow: Fetch -> Install)
│   └── hosts.ini          # Инвентарь серверов (IP узлов)
├── terraform/             # Описание облачной инфраструктуры
│   └── main.tf            # Ресурсы (VM, Network, Security Groups)
├── .github/               # CI/CD автоматизация
│   └── workflows/         # Сценарии деплоя через GitHub Actions
├── .gitignore             # Игнорируемые файлы (*.tfstate, id_rsa)
└── README.md              # Данная документация
🛠 Развертывание инфраструктуры (Terraform)
Перед началом убедитесь, что установлены переменные окружения для доступа к OpenStack (через GitHub Secrets или .env).

1. Инициализация
Скачивание необходимых провайдеров для работы с PS Cloud:

Bash

terraform init
2. Планирование
Проверка ресурсов, которые будут созданы (серверы, сети, правила доступа):

Bash

terraform plan
3. Применение
Создание ресурсов в облаке:

Bash

terraform apply -auto-approve
🚀 Конфигурация серверов (Ansible)
После того как Terraform поднял серверы, Ansible берет на себя установку софта.

🔐 Безопасность и Секреты
Проект использует GitHub Actions Secrets для защиты конфиденциальных данных. В плейбуке используются следующие переменные:

s3_host: Адрес S3 хранилища (213.155.21.230).

s3_username: Логин для доступа к MinIO.

s3_password: Пароль для скачивания ролей.

1. Проверка доступности
Проверяем SSH-соединение со всеми узлами:

Bash

ansible all -m ping -i hosts.ini
2. Запуск настройки (Fetch & Deploy)
Основной сценарий автоматически скачивает роли из S3 хранилища, распаковывает их и настраивает кластер:

Bash

ansible-playbook -i hosts.ini site.yml --extra-vars "s3_username=admin s3_password=********"

Apache: 3 узла с персонализированными страницами Welcome to <hostname>.

HAProxy: Балансировка трафика Round-Robin.

Мониторинг: Веб-интерфейс статистики доступен по адресу http://185.102.75.187:8080/stats.

Прежде чем запускать команды, нужно убедиться, что Terraform установлен. 
Инициализация  
Скачиваем необходимые драйверы (провайдеры) для работы с облаком PS Cloud:
Bash
terraform init

Планирование 
Проверяем, какие ресурсы будут созданы (серверы, сети, балансировщик):
Bash
terraform plan

Применение 
Создаем инфраструктуру в облаке:
Bash
terraform apply
(Введите yes для подтверждения)

Ansible 
 Настройка серверов
Когда серверы созданы и доступны по сети, переходим к их конфигурации. Убедитесь, что вы находитесь в корне проекта или папке ansible/.
Проверка доступности  
Убедитесь, что Ansible видит все узлы из вашего файла hosts.ini:
Bash
ansible all -m ping -i hosts.ini
Запуск настройки 
Запускаем основной сценарий (плейбук), который установит Apache, настроит HAProxy и применит политики безопасности:
Bash
ansible-playbook -i hosts.ini site.yml
